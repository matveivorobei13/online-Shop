# Online Shop API

Backend для интернет-магазина: REST API на Node.js с JWT-аутентификацией, корзиной, заказами и управлением товарами/категориями.

## Стек

- **Node.js / Express 5**
- **PostgreSQL** (`pg`) — прямые SQL-запросы, без ORM
- **JWT** (access + refresh токены) для аутентификации
- **bcrypt** — хеширование паролей
- **zod** — валидация входных данных
- **multer** — загрузка файлов (аватарки, изображения товаров)

## Архитектура

Проект построен по слоям:

```
routes/        → маршруты, привязка middleware
controllers/    → приём запроса, разбор параметров, формирование ответа
services/       → бизнес-логика
repositories/   → SQL-запросы к базе данных
middleware/     → аутентификация, валидация, загрузка файлов
schemas/        → zod-схемы валидации
```

## Установка и запуск

1. Клонировать репозиторий и установить зависимости:

```bash
git clone https://github.com/matveivorobei13/online-Shop.git
cd online-Shop
npm install
```

2. Создать файл `.env` на основе `.env.example` и заполнить переменные:

```
JWT_SECRET=     # секрет для подписи JWT
PORT=           # порт, на котором будет запущен сервер
DB_HOST=
DB_USER=
DB_PASSWORD=
DB_NAME=
DB_PORT=
FAKE_HASH=      # bcrypt-хеш произвольной строки, используется для защиты от user enumeration при логине
```

`FAKE_HASH` можно сгенерировать так:
```js
const bcrypt = require("bcrypt")
console.log(bcrypt.hashSync("anything", 10))
```

3. Поднять PostgreSQL и создать таблицы (структура — под свою БД, схема миграций пока не приложена).

4. Запустить сервер:

```bash
npm start            # обычный запуск
npx nodemon app.js   # с автоперезапуском при разработке
```

## Аутентификация

- При регистрации/логине выдаётся пара токенов: `access` (в теле ответа) и `refresh` (в httpOnly-cookie).
- `access` передаётся в заголовке `Authorization: Bearer <token>`.
- `refresh` обновляет `access` через `GET /api/auth/refresh`, не требуя повторного ввода пароля.
- Роль `admin` выдаётся только вручную через БД — через API стать админом нельзя.

## Эндпоинты

### Auth — `/api/auth`

| Метод | Путь        | Доступ      | Описание                                   |
|-------|-------------|-------------|---------------------------------------------|
| POST  | `/register` | публичный   | Регистрация (+ загрузка аватарки)            |
| POST  | `/login`    | публичный   | Вход, выдача токенов                          |
| GET   | `/me`       | авторизован | Данные текущего пользователя                  |
| GET   | `/refresh`  | публичный   | Обновление access-токена по refresh-cookie    |
| POST  | `/logout`   | публичный   | Выход, инвалидация refresh-токена             |

### Products — `/api/products`

| Метод  | Путь   | Доступ    | Описание                      |
|--------|--------|-----------|---------------------------------|
| GET    | `/`    | публичный | Список товаров                  |
| GET    | `/:id` | публичный | Товар по id                     |
| POST   | `/`    | admin     | Создать товар (+ изображение)    |
| PUT    | `/:id` | admin     | Редактировать товар               |
| DELETE | `/:id` | admin     | Удалить товар                     |

### Categories — `/api/categories`

| Метод | Путь              | Доступ    | Описание          |
|-------|-------------------|-----------|----------------------|
| GET   | `/get_categories`  | публичный | Список категорий       |
| POST  | `/create_category` | admin     | Создать категорию       |

### Cart — `/api/cart`

| Метод  | Путь   | Доступ      | Описание                       |
|--------|--------|-------------|-----------------------------------|
| GET    | `/`    | авторизован | Содержимое своей корзины           |
| POST   | `/:id` | авторизован | Добавить товар в корзину            |
| PUT    | `/:id` | авторизован | Изменить количество товара           |
| DELETE | `/:id` | авторизован | Удалить товар из корзины              |
| DELETE | `/`    | авторизован | Очистить корзину                       |

### Orders — `/api/orders`

| Метод | Путь   | Доступ      | Описание                        |
|-------|--------|-------------|------------------------------------|
| GET   | `/`    | авторизован | Список своих заказов                |
| GET   | `/:id` | авторизован | Свой заказ по id                    |
| POST  | `/`    | авторизован | Создать заказ из корзины             |
| PATCH | `/:id` | admin       | Изменить статус заказа                |

## Статусы заказа

Переходы между статусами описаны в `utils/order_statuses.js` и валидируются на уровне сервиса — сменить статус в обход разрешённых переходов нельзя.
